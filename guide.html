AnimeDL Setup Guide

01What is AnimeDL?

AnimeDL is a self-hosted web application that lets you search, browse, and download anime episodes directly to your machine. No ads, no trackers, no accounts. Everything runs locally on your own computer.

The frontend is a clean black-and-white React interface. The backend is a Python FastAPI server that talks to the Allanime API for episode metadata, then uses yt-dlp to fetch the actual video files from streaming sources.

02Prerequisites

Before you can run AnimeDL, you need these four tools installed on your system:

ToolVersionPurpose
Python3.11+Runs the FastAPI backend server
Node.js20+Runs the Vite frontend dev server
yt-dlplatestDownloads video from streaming sources
ffmpeglatestMerges separate video and audio streams (DASH)

Installation

# Python / yt-dlp
pip install yt-dlp

# ffmpeg (pick one for your OS)
sudo apt install ffmpeg          # Debian / Ubuntu
sudo pacman -S ffmpeg            # Arch Linux
brew install ffmpeg              # macOS

# Node.js — download from https://nodejs.org or use nvm
Verify

Run python3 --version, node --version, yt-dlp --version, and ffmpeg -version to confirm everything is installed.

03Quick Start

From a terminal:

# 1. Clone the repository
git clone https://github.com/wyrmic-arch/AnimeDL.git
cd AnimeDL

# 2. Install backend dependencies
cd backend
pip install -r requirements.txt
cd ..

# 3. Install frontend dependencies
cd frontend
npm install
cd ..

# 4. Start both servers
bash start.sh

Open http://localhost:5173 in your browser.

Stopping the Servers

kill $(lsof -ti:8000) $(lsof -ti:5173) 2>/dev/null
Persistent Daemons

Both servers run as daemons (via setsid) — they survive terminal disconnects. Use the kill command above to stop them.

04How to Use

The download workflow is straightforward: search, select, download, watch.

01

Search

Use the search bar on the home page, or browse trending and popular shows. Filter by genre to narrow down your options.

Every result shows the cover image, episode count, and dub availability.

02

Select

Choose Dub or Sub mode, pick your preferred quality, and select individual episodes — or use "Select All" for a full batch.

Previously downloaded episodes are marked with a check and a strikethrough.

03

Download

Hit the download button. Multiple episodes run in parallel (up to 3 at a time). Progress is shown in real time — the episode number gradually turns white.

You can stop all active downloads from the Downloads page at any time.

04

Watch

Find your files in the destination folder (default: ~/Videos/anime/). Each show gets its own subfolder; files are named by episode number.

06Tips & Notes

English Dub

English dubs are preferred when available. The Dub tab shows all episodes with English audio. The Sub tab shows original Japanese with subtitles.

Surviving Disconnects

Both servers run as daemons using setsid. You can close the terminal or SSH session — the servers keep running. Use the kill command to stop them.

Resume Downloads

If a download is interrupted (e.g., you stop all downloads or the server restarts), yt-dlp will resume partial files on the next attempt. No need to re-download from scratch.

History Persistence

Download history and search history are stored in your browser's localStorage. Clearing your browser data will clear it. The History page lets you pick up where you left off on any show.

Source Priority

The backend tries multiple source servers for each episode. Direct links from tools.fast4speed.rsvp are preferred (fastest and most reliable). If a source fails, the next one is tried automatically.

07API Reference

The backend exposes a REST API and a WebSocket endpoint. All endpoints are prefixed with /api.

MethodPathDescription
GET/api/search?q=&genres=&sort_by=Search anime by name and genre
GET/api/trendingGet trending shows (24 results)
GET/api/popularGet most popular shows (24 results)
GET/api/anime/{id}Full show detail with episodes, description, genres
GET/api/anime/{id}/sources/{ep}?mode=Source URLs for one episode
POST/api/downloadStart a download task
GET/api/download/{task_id}Check task status
GET/api/download/{task_id}/fileServe completed video file
GET/api/downloadsList all download tasks
POST/api/downloads/stopStop all active downloads
WS/ws/download/{task_id}Real-time download progress stream

WebSocket Messages

Each message is a JSON object with the task's current status, progress percentage, download speed, and ETA. The stream stays open until the download finishes or fails.
